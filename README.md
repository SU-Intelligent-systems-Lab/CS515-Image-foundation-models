# DINO-VPR: A Study of DINO Vision Foundation Models and Their Adaptation to Visual Place Recognition

> **Course project.** This repository accompanies a report studying the DINO
> family of self-supervised vision foundation models (DINO, DINOv2, DINOv3)
> and their adaptation to the Visual Place Recognition (VPR) task.

---

## Project overview

The project is organised in three parts, each producing a concrete deliverable:

| Part | Content | Deliverable |
|------|---------|-------------|
| **I**  | Theoretical study of the DINO family | `report/report.pdf` — Chapters 1–3 |
| **II** | Feature analysis of pretrained DINOv2 + mini from-scratch reimplementation of the DINO training loop on a small dataset | Notebooks `01_*` and `02_*`, populated Chapter 3, report Chapter 4 |
| **III**| VPR adaptation experiments: frozen baseline, LoRA-adapted backbone, EffoVPR-style training-free re-ranker | Notebook `03_*`, report Chapters 5–8 |

The goal is **not** to compete with the public DINOv2/v3 checkpoints — training
at that scale is infeasible for a course project. Instead, we (a) demonstrate
understanding of the training mechanism through a small-scale reimplementation
and (b) use the public checkpoints as a frozen backbone to build and benchmark
a real VPR system.

## Installation

Tested on Python 3.10+, CUDA 11.8 / 12.1, PyTorch 2.1+. A single GPU with
12–24 GB of VRAM is assumed (e.g. RTX 3090 / 4090, Colab A100, T4).

```bash
git clone https://github.com/<your-username>/dino-vpr-project.git
cd dino-vpr-project
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -e .                 # install the `dinovpr` package editable
# --- or, equivalently, ---
pip install -r requirements.txt
```

PyTorch itself should be installed **first** with the CUDA build matching your
system, following the instructions at <https://pytorch.org/get-started/locally/>.

## Quick start

### Run the feature-exploration notebook

No training required; uses the public DINOv2 checkpoint.

```bash
jupyter lab notebooks/01_dino_feature_exploration.ipynb
```

### Reproduce the mini DINO training on CIFAR-100

```bash
python scripts/run_mini_dino.py --config configs/mini_dino_cifar100.yaml
```

Logs, checkpoints and loss curves land in `results/logs/mini_dino/`.

### Run unit tests

```bash
pytest tests/ -v
```

---

## Reproducibility

Every experiment is parameterised by a YAML config in `configs/` and a
fixed random seed. All figures and tables in the report are generated by
scripts committed here. A `results/` folder snapshotted at submission time
is archived alongside the report.
